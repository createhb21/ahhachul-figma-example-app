## 2026-03-22

- 단일 데모 JSX를 Vite 앱으로 포장할 때는 원본 파일을 그대로 보존하고 `src/` 아래 복제본을 두는 편이 검증과 비교가 쉽다.
- 모바일 셸로 감쌀 때 루트와 고정 하단 바의 `width: 768` 같은 절대 폭은 `width: 100%` + `maxWidth`로 먼저 치환해야 작은 화면에서 가로 스크롤이 생기지 않는다.
- 패키징 완료 전에는 `vite preview`에 Playwright 스모크 테스트를 붙여 초기 렌더, 콘솔 에러, 핵심 탭 전환까지 실제 브라우저로 확인하는 편이 빠르고 확실하다.
- 빈 원격 저장소에 첫 커밋을 넣을 때는 로컬 프로토타입 디렉터리를 통째로 복사하기보다 `src`, `public`, 설정 파일, 문서만 선별 복사해야 템플릿 찌꺼기와 빌드 산출물을 함께 올리지 않게 된다.
- 단일 파일 앱을 라우터로 쪼갤 때는 먼저 홈 내부 상태로 묶여 있던 화면(예: 검색)을 실제 URL로 끌어낸 뒤, route wrapper를 lazy import 해서 브라우저 검증과 chunk 출력을 함께 보는 편이 안전하다.
- 기존 코드가 `React.useState/useEffect/useRef` 패턴을 섞어 쓰고 있다면 route 분리 전에 반드시 `React` namespace import 유무를 확인해야 한다. 기존에는 안 터지던 경로도 lazy route에서 바로 런타임 에러로 드러날 수 있다.
